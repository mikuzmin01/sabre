вопрос зачем нужен AppConfig и на что влияет название в списке приложений 'catalog.apps.CatalogConfig' или 'catalog'
 ответ это возможность подключения приложения в INSTALLED_APPS с прописанным AppConfig, можно сделать это в самом файле AppConfig (и даже много больше) или 
в __init__.py прописать список стартующих приложений default_app_config = 'catalog.apps.CatalogConfig'.
Настраиваемые атрибуты  
AppConfig.name например 'django.contrib.admin' путь для импорта приложения Должен быть уникальным для проекта.
AppConfig.verbose_name  Читабельное название приложения По умолчанию равен label.title().
AppConfig.path  Путь в файловой системе к каталогу с приложением
	Методы
		AppConfig.get_models()
    Возвращает итератор по классам Model.

		AppConfig.get_model(model_name)
    Возвращает Model для переданного model_name. Вызывает LookupError, если модель не существует. model_name регистро-независимое значение.

		AppConfig.ready()
    Класс наследник может переопределить этот метод, чтобы выполнить инициализацию приложения, например зарегистрировать сигналы. 
	Вызывает, когда реестр приложений будет полностью инициализирован и все приложения будут добавлены.
    Вы не можете импортировать модели в модуле, который содержит классы настроек, но вы можете использовать метод get_model(),
	чтобы получить модель по названию:

    def ready(self):
        MyModel = self.get_model('MyModel')
